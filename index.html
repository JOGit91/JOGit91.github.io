What I Do Core Focus

Detection Engineering at Scale

I lead and contribute to a detection engineering team responsible for 945+ active production detections spanning 97 log types from 40+ vendor integrations (Windows, macOS, Linux, M365, Azure/Entra ID, AWS, GCP, Google Workspace, EDR, firewalls, email, identity, and more). I remain one of the team's most active individual contributors, personally authoring detections and handling complex threat modeling for high-priority coverage areas. I use MITRE ATT&CK to identify coverage gaps and prioritize development using a risk-based approach.

Team Leadership & Agile Delivery

As Scrum Lead for the detection engineering team, I own the full agile delivery cycle, running sprint planning, backlog refinement, and retrospectives, setting quarterly OKRs, and tracking team metrics like velocity and sprint completion. I also manage two direct reports and maintain regular 1:1s focused on growth and performance.

Detection-as-Code

I co-designed and implemented a scalable detection-as-code platform that replaced manual, ungoverned detection management with version-controlled YAML definitions, CI/CD validation pipelines, automated cloud deployment, and peer code review. Every detection change is tracked, tested, and reviewed before it reaches production.

Detection Health & Metrics

I built a comprehensive detection lifecycle monitoring program that tracks precision, false positive rates, MTTD, DRAPE scores (Detection Reliability And Precision Efficiency), match dropoff/spikes, resolution trends, and customer filter activity. This data drives every tuning and prioritization decision we make.

GenAI Applied to Security Operations

I apply GenAI (Cursor, Claude Code) to build production tooling, automate investigation playbook generation, and have designed evaluation frameworks for LLM-powered security features including scoring criteria for accuracy, groundedness, relevance, and temperance.

Detection Suite Performance Selected Work

Over the past two years, the detection engineering team has made significant measurable improvements to detection quality, including meaningful gains in precision, reductions in mean time to detect for high-priority threats, and steady growth in detection coverage across customer environments. I track these outcomes through a custom reporting program built in BigQuery, covering precision rates, false positive trends, MTTD by detection type, DRAPE scores, and customer coverage metrics. Data drives every prioritization and tuning decision we make.

Tooling & Automation Selected Work

I systematically automate process overhead so the team can focus on high-judgment detection engineering work.

DETAILR

Production test analysis suite that ingests raw test output and provides statistical summaries, per-org match breakdowns, field-level frequency analysis, and cooldown window simulation to predict finding volume before deployment. Now standard tooling for every detection release.

Workflow Generation System

GenAI-powered investigation playbook generator aligned to NIST 800-61r2 incident response phases. Reduced workflow creation time from 1-2 hours to minutes while raising quality consistency across the team. The result is more accurate, consistent playbooks that guide responders of any experience level through an incident with confidence.

AutoDF

Automated the process of applying detection filters at scale, replacing what was previously a time-consuming, manual, org-by-org process with a single, dedicated tool.

Sprint Planning Automation

Data-driven sprint planning pipeline that cross-references Jira tickets against BigQuery/HubSpot data (ACV, health scores, at-risk flags) to surface customer signals and enforce prioritization frameworks.

MR Pre-Review Automation

Mined 860+ merged MRs to identify patterns in review feedback, then built automated pre-review checks to catch mechanical issues before human reviewers, letting them focus on detection logic quality and threat modeling.

Detection Knowledge Assistant

An LLM-powered query interface built on top of the detection-as-code repository. It lets anyone ask natural-language questions about detection coverage and get accurate, metadata-backed answers instantly, making the detection library accessible to the whole organization, not just the engineers who built it.

Projects Side Projects

M365 Identity Threat Codex

A reference resource documenting attack vectors targeting Microsoft 365 environments, built for defenders and administrators. Each entry covers how the attack works, evidence indicators, relevant threat actor profiles, KQL hunt queries optimized for Microsoft Sentinel and Defender XDR, and prioritized mitigations. The Hardening Matrix lets defenders sort controls by impact or ease of implementation, select what fits their environment, and export a structured, stakeholder-ready implementation plan.

View Codex View on GitHub

SENTINEL — Threat Intelligence Dashboard

A self-built, fully static threat intelligence platform that continuously ingests security news, uses Claude to extract and structure threat actor and TTP data, and tracks active threat actors, trending tactics, and tooling over time with historical activity charts. It includes an Attribution tool that matches observed indicators against tracked actor profiles, and an Org Risk Profile tool that maps an organization's industry, cloud stack, and infrastructure to relevant threat actors, trending tactics, recommended hunts, and prioritized mitigations. Built entirely on GitHub Pages with GitHub Actions automation handling ingestion and enrichment on a recurring schedule.
